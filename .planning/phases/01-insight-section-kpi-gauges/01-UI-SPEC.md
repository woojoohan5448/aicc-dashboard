## Spacing Scale

All declared tokens are multiples of 4. Existing codebase values that are not multiples of 4 (e.g., `.box` padding 22px, `.g3` gap 18px) are consumed via class reuse but are NOT declared as spacing tokens in this contract.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Inline label gaps |
| sm | 8px | Compact element spacing, tag padding |
| md | 16px | Default element spacing, grid gaps within cards |
| lg | 24px | Card internal padding (section-level internal padding) |
| xl | 32px | Page-level horizontal padding (matches `.page` padding) |
| section-gap | 24px | Vertical gap between insight section and preceding content |

**Passthrough (class reuse, not new tokens):** The `.box` class applies 22px padding and `.g3` applies 18px gap. These values are inherited from existing CSS classes and are not declared as spacing tokens. New elements in this phase that need explicit spacing must use the token values above.

## Component Inventory

### 1. Insight Section Container

**HTML structure:**
```
div.stitle  -->  "KPI 달성 현황" section title
div.g3      -->  3-column grid holding 3 gauge cards
```

**Placement:** Immediately after the last existing section in `buildOv()` (after Top10 grid), before the closing backtick of `el.innerHTML`.

**CSS classes:** Reuse existing `.stitle` and `.g3` classes. No new container class needed.

**Section title text:** `"KPI 달성 현황"` with subtitle span `"목표 대비 현재 달성률"`

**Section title icon:** `"🎯"` (target emoji, consistent with emoji-as-icon pattern)

### 2. Gauge Card (x3)

**Container:** Reuse existing `.box` class (white card, 12px border-radius, existing padding, shadow).

**Internal layout (top to bottom):**

| Element | Class | Style |
|---------|-------|-------|
| KPI name label | `.sb-label` (reuse) | `font-size:12px; color:var(--dim); font-weight:700; margin-bottom:12px;` |
| Canvas gauge | new `<canvas>` element | `width:120px; height:120px; margin:0 auto 12px;` |
| Achievement value | `.sb-val` (reuse) | `text-align:center; font-size:28px;` with unit span |
| Target context | `.sb-tag` (reuse) | Centered below value |

**Canvas gauge specifications:**

| Property | Value |
|----------|-------|
| Canvas size | 120 x 120 CSS pixels (240 x 240 device pixels at 2x) |
| Arc type | Semi-circle (180 degrees, from PI to 0) |
| Arc center | (60, 70) -- offset down to leave room for percentage text above |
| Arc radius | 48px |
| Arc line width | 10px |
| Arc line cap | `round` |
| Track arc | Full semi-circle in `var(--border)` color |
| Filled arc | Proportional to achievement rate, in `var(--pink)` color |
| Center text | Achievement percentage (e.g., "103%") in 15px bold DM Mono, color `var(--text)` |

**Achievement rate calculation per KPI type:**

| KPI ID | Type | Formula | Display |
|--------|------|---------|---------|
| AICC_RRMS | absolute | `current / target` | `(current/target*100).toFixed(0) + '%'` |
| AICC_KA_REF | count | `current / target` | `(current/target*100).toFixed(0) + '%'` |
| AICC_BUILD_ORDER | achievement | `current / target` | `(current/target*100).toFixed(0) + '%'` |

**Value display below gauge:**

| KPI | Value format | Unit | Tag text |
|-----|-------------|------|----------|
| AICC RRMS | `current.toFixed(2)` (e.g., "0.62") | `%` | `"목표 " + target.toFixed(2) + "%"` |
| KA 수주 레퍼런스 | `current` (e.g., "6") | `건` | `"목표 " + target + "건"` |
| 구축형 수주액 | `fmt(current)` (e.g., "330") | `억` | `"목표 " + fmt(target) + "억"` |

### 3. No New Chart.js Instances

Gauges are rendered with raw Canvas 2D API, not Chart.js. Reason: simple semi-circle arcs do not justify a Chart.js doughnut config. This keeps the implementation lightweight and avoids chart lifecycle management overhead.

The gauge render function should be a standalone utility: `drawGauge(canvasId, ratio, color)`.

## Implementation Notes

1. **Scope constraint:** All changes inside `buildOv()` function only. The insight section HTML is appended to the template literal that builds `p-ov` content.
2. **CSS additions:** Minimal. Reuse `.stitle`, `.g3`, `.box`, `.sb-label`, `.sb-val`, `.sb-tag`. Only new CSS needed is for canvas centering within the card (inline style sufficient).
3. **No new dependencies:** Canvas 2D API is native. No Chart.js gauge plugin needed.
4. **Gauge utility function:** Define `drawGauge(canvasId, ratio, color)` inside `buildOv()` or as a module-level function. Called 3 times, once per KPI item.
5. **Data source:** `D.summary.kpi` for base_period; `D.kpi.items` array for the 3 KPI objects with `target`, `current`, `type` fields.

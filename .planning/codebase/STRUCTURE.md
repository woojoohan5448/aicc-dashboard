## Naming Conventions

**Files:**
- `index.html`: Main application file
- `index_backup.html`: Previous version snapshot
- `AICC_대시보드_v22.html`: Version archive (Korean naming)
- Versioning in git commits only (v22, v27, v28)

**HTML Element IDs:**
- Pages: `p-{pagename}` (p-ov, p-sub, p-bld, p-ch, p-comp, p-kpi, p-edit)
- Charts: `{page}-{subject}` (ov-rev, sub-monthly, bld-timeline)
- Data containers: Semantic names (ov-sub-pie-legend, proj-list, ind-table)
- Toast notification: `#toast` with child elements (t-icon, t-msg, t-sub)

**CSS Classes:**
- Utility classes: `.g2`, `.g3`, `.g4` (grid layouts)
- Component classes: `.box`, `.kcard`, `.kpi-card-new`, `.sb-group`
- Modifier classes: `.on` (active state), `.show` (visible state)
- Semantic naming: `.edit-section`, `.edit-sh`, `.edit-body`, `.edit-grid2`
- Status classes: `.tb.on` (active tab), `.page.on` (visible page)

**JavaScript Functions:**
- Page builders: `build{PageName}()` (buildOv, buildSub, buildBld, buildCh, buildComp, buildKpi, buildEdit)
- Utilities: camelCase (fmt, fmtK, fmtP, toast, go, dc, rebuildAll)
- Event handlers: Inline in HTML or registered in functions
- Constants: UPPERCASE (SEED, D, PINK, PD, BLUE, GREEN, AMBER, GRAY)

**Variables:**
- Data objects: PascalCase (SEED, D, Chart) or abbreviated (kpi, an, sub, ch, mon)
- Loop indices: Single letter (i, j, k, v)
- Temporary: Short-lived descriptors (ctx, cumIdx, lineX, barMeta)

**CSS Custom Properties:**
- Color theme: `--pink`, `--blue`, `--green`, `--red`, `--amber`
- Semantic: `--bg`, `--card`, `--text`, `--dim`, `--muted`, `--border`
- Shadow: `--shadow`, `--shadow-md`

## Where to Add New Code

**New Feature:**
- Primary code: Add in `index.html` between `<script>` tags (after line 3538)
- Function structure: Follow `build{Name}()` pattern for page builders
- Data: Add properties to SEED object (lines 3540-3550)
- HTML template: Add new page `<div class="page" id="p-{name}"></div>` in HTML section
- CSS: Add styles in `<style>` tag (after line 435, before closing tag)
- Tab button: Add in `.tabs` section (after line 420)
- Event handler: Register in function or attach via onclick attribute

**New Component/Module:**
- Implementation: Create class or function in JavaScript section
- Naming: Use descriptive camelCase for functions (e.g., `buildProductChart()`)
- Pattern: Accept container element ID as parameter, manipulate via getElementById
- Chart integration: Use `dc(id)` pattern for lifecycle management
- Data access: Read from D object, support mutations for edit tab

**Utilities:**
- Shared helpers: Place near top of JavaScript section (after constants, before page builders)
- Formatting: Add to fmt/fmtK/fmtP family
- Examples: `fmtCurrency()`, `fmtDate()`, `fmtPercent()`

**Charts:**
- Add new Chart.js instance in appropriate build function
- Register in charts object: `charts[id] = new Chart(...)`
- Destroy existing: Call `dc(id)` before creating
- Use global Chart.defaults for consistency
- Plugin patterns: See label plugins in buildOv (lines 3843-3930)

**CSS Styling:**
- Follow existing patterns for spacing (12/16/20/24px)
- Use CSS variables for colors
- Grid layouts: .g2 (2-col), .g3 (3-col), .g4 (4-col)
- Cards: `.box` base class for white card with border
- Status colors: Use predefined classes (.bg-p, .bg-u, .bg-d, .bg-n)

**Data Structure:**
- Follow SEED object nesting pattern
- Annual data: `{ labels: [...], metric: [...] }`
- Monthly data: `{ labels: [...], data: { series1: [...], series2: [...] } }`
- Top-N lists: Array of `{ name, value }` or `{ name, value, category }`
- Deep copy before mutation: `D = JSON.parse(JSON.stringify(SEED))`
